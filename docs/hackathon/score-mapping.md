# 评分点映射

## 功能实现

- 基础需求执行：共享任务编排器可输出并执行结构化计划
- 显式意图识别：`TaskPlan.intent_label` 默认由模型语义规划产出，本地仅保留通用观测标签兜底
- 信息不足澄清：`TaskPlan.response_mode=clarify` 支持在参数不全时先补充信息再执行
- 高风险识别与处置：`RiskEvaluator` 支持阻断、确认和解释
- 复杂连续任务处理：`TaskPlan.steps` 支持多步拆解，并在执行中通过 `reflect -> revise / recover / continue` 调整后续步骤

## 操作环境感知与决策

- 环境信息感知：执行前读取系统信息并归纳发行版、主机名、包管理器
- 环境工具选择依据：`TaskStep.environment_rationale` 显式说明为何在当前 Linux 环境选择该命令
- 基于环境的安全判断：关键路径、敏感文件、权限操作按 Linux 场景判断
- 持续状态更新与决策：CLI 主链在每步执行后都会记录观察结果、反思决策、计划调整或恢复建议

## 结果反馈与执行说明

- 解析链清晰度：`TaskPlan.parsing_summary` 将输入理解、任务拆解、环境决策、风险处置、结果模式结构化呈现
- 执行反馈清晰度：步骤级输出、分析结果、最终反馈统一展示
- 风险处置依据说明：每个风险步骤都有 `RiskAssessment.explanation`
- 命令选择依据说明：每个步骤可携带 `selection_reason`
- 交互反馈连贯性：CLI 执行结果会连续展示 `step -> analysis -> reflection -> next_action`

## 操作闭环完整性

- 单轮闭环：自然语言 -> 计划 -> 风险 -> 执行 -> 反馈
- 澄清闭环：自然语言 -> 识别信息不足 -> 输出补充问题 -> 等待继续执行
- 风险场景闭环：阻断/确认/拒绝后仍保留完整轨迹
- 连续任务闭环稳定性：失败时优先进入恢复链，成功时可按观察结果微调剩余步骤

## 性能与稳定性

- 无数据库依赖，部署轻量
- 审计文件直接落盘，便于回放
- 保留旧流程兜底，降低改造风险
