# 赛题测试案例手册

本文档面向 AI Hackathon 2026 赛题要求编写，目标是从赛题视角验证当前 OS_Agent 已满足、部分满足和尚未满足的目标。

定位说明：
- 本文档是“赛题验收案例手册”，用于人工验证、录制演示视频和答辩准备。
- [self-test.md](/home/shl203/桌面/OS_Agent/docs/hackathon/self-test.md) 继续承担“快速启动与自测说明”职责。
- [tests/test.md](/home/shl203/桌面/OS_Agent/tests/test.md) 继续承担“程序侧手测脚本”职责。

## 使用方式

建议在真实 Linux 环境中执行，并至少保留以下可观察证据：
- CLI 交互输入与输出
- `~/.os_agent/tasks` 下的 `request.json`、`plan.json`、`result.json`、`events.jsonl`
- 关键失败场景下的系统命令输出与日志

建议通用前置条件：
- 已安装依赖：`pip install -r requirements.txt`
- 已配置模型访问参数：`config.yaml`
- CLI 启动命令：`python3 os_agent.py -c config.yaml`

当前判定标准：
- `满足`：当前程序已有明确执行路径，且可提供赛题 4.3 要求的可观察证据
- `部分满足`：已有实现基础，但覆盖面、稳定性或交互形态仍有限
- `未满足`：当前无实际验证路径，或能力尚未落地
