# Prompt 与工具说明

## Agent 能力边界

OS_Agent 不做单纯请求透传，而是进行以下本地决策：

1. 读取目标 Linux 环境信息
2. 识别输入属于查询、创建、删除、诊断或通用执行类任务
3. 对信息不足的请求进入 `clarify` 分支，而不是直接生成命令
4. 让 LLM 生成命令或多步命令计划
5. 对每一步补充命令选择依据、环境选择依据和预期结果
6. 对高风险步骤进行确认或阻断
7. 执行命令并分析结果
8. 将执行过程和结果写入审计文件

## Prompt 设计原则

当前项目沿用现有 provider 中的命令生成与结果分析 Prompt，并在编排层补上结构化约束：

- 优先返回显式 `intent_label`，且标签允许由模型按任务语义自由命名，而不是限制在固定规则枚举中
- 信息不足时返回 `clarify` 而不是猜测执行
- 优先生成能在当前系统直接执行的命令
- 输出需可拆分为单步或多步计划
- 每一步应尽量携带 `expected_outcome`、`selection_reason`、`environment_rationale`
- 命令不能脱离当前 Linux 环境
- 风险命令必须可解释
- 命令执行失败时必须输出可读的原因与建议

## 工具定义

### 命令执行器

- 实现：`src/executors/linux_command.py`
- 能力：执行 Linux 命令、采集系统信息、处理交互式编辑器命令

### 任务编排器

- 实现：`src/service/orchestrator.py`
- 能力：意图识别、澄清分流、命令拆步、环境决策说明、风险门控、执行轨迹生成

### 风险评估器

- 实现：`src/service/risk.py`
- 能力：识别高风险路径、敏感文件、用户权限管理、系统级变更

### 审计存储

- 实现：`src/service/audit.py`
- 能力：落盘请求、计划、事件和结果

### CLI 交互层

- 实现：`src/agent.py`
- 能力：承接终端输入、驱动任务编排器执行并展示结果
